# Development Guidelines

## Git Commit Guidelines
- Do not include tool or AI assistant (CLAUDE) references in GitHub commit comments
- Keep commit messages focused on technical changes and business value

## Repository Management
- Do not commit or push to GitHub unless explicitly requested
- Review all staged changes before committing
- Ensure all tests pass before pushing

## Code Quality
- Follow established patterns in the codebase
- Maintain consistent formatting and style
- Update documentation when making changes

---

*These guidelines help maintain professional standards in the EO Framework Solutions repository.*
